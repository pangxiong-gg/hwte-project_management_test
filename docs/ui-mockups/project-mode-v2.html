三種模式總結比較

瀑布式

  • 看板：5 個階段欄位（線性）
  • 任務類型：依階段嚴格限制
  • 適合：需求明確、變更少
  • 風險：階段僵化、回頭成本高

敏捷式

  • 看板：Sprint 循環欄位
  • 任務類型：完全自由
  • 適合：需求變動頻繁
  • 風險：缺乏大方向管控

混合型

  • 看板：大階段 + Sprint 雙層
  • 任務類型：前期限制、後期自由
  • 適合：中大型專案
  • 風險：複雜度較高

系統實作重點

  • 建立專案時選擇模式（不可更改）
  • 依模式動態生成看板欄位
  • 依階段動態限制任務類型選項
  • 不同模式顯示不同報表/儀表板
