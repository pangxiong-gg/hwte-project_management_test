# 討論記錄 #2：實作進度回顧

## 日期
2026-06-01（更新）

## 目前已完成

### 後端
- JWT 認證系統（登入/登出/Token 驗證）
- CRUD API：專案 / 需求 / 任務 / Bug / 測試用例 / 用戶 / 通知
- Prisma Schema（10 個模型：User, Project, Requirement, ProjectPhase, Task, TestCase, TestExecution, Bug, Notification, RequirementChange）
- 角色權限中間件（roleMiddleware, adminMiddleware）
- 通知服務（任務指派/狀態變更/需求變更自動通知）
- SQLite 資料庫 + Seed 資料

### 前端
- 登入頁面
- Dashboard 佈局（側邊欄 + Header + 內容區 + 頭像下拉選單）
- 專案總覽（列表 + 新增專案表單 + 模式選擇）
- 專案詳情（基本資訊 + 統計卡片 + 階段流程 + Tab 切換顯示需求/任務/Bug/測試）
- 任務看板（Kanban，模式感知：瀑布按階段分組，敏捷/混合按狀態分組）
- 用戶管理頁面（ADMIN 專用：列表 + 角色/狀態編輯 + 新增用戶）
- 個人資料頁面（修改姓名/密碼）

## 系統已可執行的操作

1. 登入系統（依角色顯示不同權限）
2. 查看專案列表（顯示模式標籤）
3. 新增專案（選擇瀑布/敏捷/混合模式，自動生成階段）
4. 查看專案詳情（含需求/任務/Bug/測試 Tab）
5. 管理需求（新增/編輯/刪除，變更時自動記錄歷史）
6. 管理任務（看板拖曳、指派、Git 關聯 branch/commit/PR）
7. 管理 Bug（新增/指派/狀態變更）
8. 管理測試用例（新增/執行/記錄結果）
9. 管理用戶（ADMIN：創建/編輯/刪除；所有人：查看個人資料）
10. 接收通知（指派/狀態變更時自動推送）
